# Git Policy

> 知识库用 Git 管理，方便回溯和协作。

## 1. 提交规范

- `feat:` 新增/消化一篇 wiki 文章
- `docs:` 更新 index.md、说明文档
- `fix:` 修复死链、frontmatter、路径
- `refactor:` 重组分类/重命名/合并
- `chore:` 模板、配置、skills 调整

## 2. 提交内容

建议提交：
- `AGENTS.md`
- `AI_CONFIG.md`
- `wiki/`
- `docs/`

谨慎提交：
- 大体积 `assets/`

避免提交：
- 临时导出文件
- 日志中的敏感内容

## 3. 分支建议

- `main`：稳定版本
- 整理工作可在主分支直接提交，保持简洁
